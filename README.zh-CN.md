# 围棋入门闯关小游戏 · Learn to Play Go

一个通过短关卡和即时反馈，从“认识气”逐步教到完成 9 路对局的浏览器围棋入门游戏。

[English](./README.md) | [简体中文](./README.zh-CN.md)

## 为什么做这个项目

传统围棋教材对完全零基础的玩家容易显得抽象。本项目把学习路径拆成短关卡，并提供可视化反馈、大白话术语解释和经过测试验证的正解。目标很直接：完成全部课程后，玩家能够理解基本规则并下完一盘入门对局。

## 主要功能

- 七章渐进课程，覆盖气、提子、手筋、死活、围地和数子
- 使用 SVG 绘制 9 路、13 路与 19 路棋盘
- 对吃子、禁入点和送叫吃风险提供上下文提示
- 教学文案内置可点击的大白话术语词典
- 支持战术题的多步正解树
- 支持区域计分和归属可视化
- 本地保存进度、星级、音效设置，并提供自由对弈
- 内置无需联网和机器学习模型的轻量规则机器人

## 技术栈

React 19 · TypeScript · Vite · SVG · Vitest · Web Audio · localStorage

## 快速开始

```bash
npm install
npm run dev
```

## 项目结构

```text
src/
├── engine/       # 不可变规则、死活、计分和机器人逻辑
├── levels/       # 关卡数据、目标类型与正解校验
├── game/         # 单关状态机
├── components/   # 棋盘、选关、关卡、词典与自由对弈
├── data/         # 围棋术语
├── audio/        # 合成音效
└── storage/      # 进度与设置
```

## 设计原则

- 保持纯前端：不依赖账号、后端或在线服务。
- 只提示有信息价值的内容，不高亮全部合法点。
- 规则引擎保持不可变，便于安全测试和回放。
- 每个新增关卡都需要自动化正解校验。
- 对手保持轻量规则驱动，不引入神经网络模型。
